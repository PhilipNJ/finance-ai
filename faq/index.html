    Frequently Asked Questions (FAQ)¶

    Common questions about Finance AI Dashboard.

    General¶

    What is Finance AI Dashboard?¶

    Finance AI Dashboard is an offline personal finance manager that uses local AI to automatically parse, categorize, and visualize your financial transactions from bank statements, credit card statements, and other financial documents.

    Key Features: - ü§ñ AI-powered document understanding - üîí 100% offline and private - üìä Automatic categorization - üìà Interactive visualizations - üíæ Dynamic database schema

    Is my financial data safe?¶

    Yes - 100% safe and private!

    Your data never leaves your computer: - ‚úÖ All processing happens locally - ‚úÖ No internet connection required - ‚úÖ No cloud uploads - ‚úÖ No external API calls - ‚úÖ Open source code (you can audit it)

    graph LR
    A[Your Files] --> B[Your Computer]
    B --> C[Local AI]
    C --> D[Local Database]

    B -.->|Never| E[‚ùå Internet]

    style E fill:#ffcdd2

    Do I need an internet connection?¶

    No! Finance AI works completely offline after initial setup.

    Internet only needed for: - Downloading the AI model (one-time, 4.7GB) - Installing dependencies (one-time)

    After setup: - ‚úÖ Process files offline - ‚úÖ View dashboard offline - ‚úÖ Everything works offline

    Is it free?¶

    Yes! Finance AI is completely free and open source.

    • ‚úÖ No subscription fees
    • ‚úÖ No hidden costs
    • ‚úÖ No "premium" features
    • ‚úÖ MIT License (use freely)

    What makes this "AI-first"?¶

    Unlike traditional finance apps that use keyword matching:

    Traditional Apps: 

    IF description contains "Whole Foods" THEN category = "Groceries"

    Finance AI: 

    AI understands: "WHOLEFDS #123" ‚Üí Whole Foods ‚Üí Grocery store ‚Üí Groceries

    The AI understands context, not just keywords!

    Installation & Setup¶

    What are the system requirements?¶

    Minimum: - Python 3.10+ - 6GB RAM - 5GB free disk space - macOS, Linux, or Windows

    Recommended: - Python 3.11+ - 8GB+ RAM - SSD storage - 4+ CPU cores

    How long does installation take?¶

    Time Breakdown:

    Step Duration
    Download AI model 5-10 min (4.7GB)
    Install dependencies 2-5 min
    First app startup 5-10 sec
    Total ~15 minutes

    Subsequent startups: 2-5 seconds

    Can I use a different AI model?¶

    Yes! You can use any GGUF-format model.

    Popular alternatives:

    Model Size Speed Quality
    TinyLlama-1.1B 600MB Fast Basic
    Mistral-7B (default) 4.7GB Medium Good
    Llama-2-13B 13GB Slow Best

    How to switch:

    Edit llm_handler.py: 

    model_path = Path("your-model-name.gguf")

    See: LLM Integration Guide

    Do I need a GPU?¶

    No! Finance AI works fine on CPU.

    Performance:

    Hardware Processing Time
    CPU only 2-5 seconds
    M1/M2 Mac (Metal) 1-2 seconds
    NVIDIA GPU 1-2 seconds

    GPU support (optional): - macOS: Metal (M1/M2) - Linux: CUDA (NVIDIA) - Windows: CUDA (NVIDIA)

    Improves over time: 1. You upload transactions 2. AI guesses categories (70-80% right) 3. You correct wrong ones 4. AI learns from corrections 5. Next upload is more accurate!

    Technical Questions¶

    How does the AI work?¶

    Three-Agent System:

    graph LR
    A[Upload File] --> B[Agent 1: Extract]
    B --> C[Agent 2: Organize]
    C --> D[Agent 3: Database]
    D --> E[Dashboard]

    style B fill:#e1f5ff
    style C fill:#e8f5e9
    style D fill:#fff3e0
    1. Agent 1 (Extractor): Reads file and extracts raw data
    2. Agent 2 (Organizer): Structures data into transactions
    3. Agent 3 (Database): Creates/updates database schema and stores records

    See: AI Architecture | Agent Workflow

    What is dynamic schema evolution?¶

    The database adapts to your data automatically.

    Traditional Database: 

    ‚ùå Must define schema first
‚ùå ALTER TABLE manually for new fields
‚ùå Rigid structure

    Finance AI: 

    ‚úÖ Tables created on-demand
‚úÖ Columns added automatically
‚úÖ Flexible structure

    Example:

    Upload a budget file ‚Üí budgets table created automatically! 

    {"month": "2025-01", "category": "Groceries", "planned": 500}
    ‚Üì 
    CREATE TABLE budgets (month TEXT, category TEXT, planned REAL);

    See: Database Schema

    Location: data/finance.db

    Tables: - transactions - Financial transactions - mem_labels - Learned categorization rules - Dynamic tables - Created based on uploaded data

    Size: ~100KB per 1,000 transactions

    Troubleshooting¶

    Why is processing slow?¶

    Common causes:

    1. First upload: Model loads (5-10 sec) - normal!
    2. Large file: >10MB takes longer
    3. Low RAM: Close other apps
    4. CPU threads: Adjust in llm_handler.py

    See: Troubleshooting: Slow Processing

    Why aren't my transactions showing?¶

    Check:

    1. ‚úÖ Date range filter (expand to "All Time")
    2. ‚úÖ Category filters (check all categories)
    3. ‚úÖ Database query: sqlite3 data/finance.db "SELECT COUNT(*) FROM transactions;"

    See: Troubleshooting: Transactions Not Appearing

    The categories are wrong!¶

    This is normal! The AI learns over time.

    Fix: 1. Click transaction 2. Change category 3. AI remembers for next time

    After 3-5 uploads: Accuracy improves to 90-95%!

    Can I reset everything?¶

    Yes! To start fresh:

    # ‚ö†Ô∏è Backup first!

# Restart app (creates new database)

    Advanced Usage¶

    Can I automate uploads?¶

    Yes! Use Python scripting:

    from finance_ai.agents import AgentWorkflow

workflow = AgentWorkflow()

# Process multiple files
files = ['jan.csv', 'feb.csv', 'mar.csv']
for file in files:
    result = workflow.process_file(file, file)
    print(f"{file}: {result['records']} records")

    Scheduled uploads:

    # Cron job (Linux/Mac) - daily at 8am
0 8 * * * cd /path/to/finance-ai && .venv/bin/python batch_upload.py

    Can I query data with Python?¶

    Yes! Use pandas:

    import sqlite3
import pandas as pd

# Connect to database
conn = sqlite3.connect('data/finance.db')

# Query transactions
df = pd.read_sql_query("""
    SELECT * FROM transactions 
    WHERE date >= '2025-01-01'
""", conn)

# Analyze
spending = df[df['amount'] < 0].groupby('category')['amount'].sum()
print(spending)
